# World Cup Predictor Lab

A Next.js football analytics lab for the **2026 FIFA World Cup**. Build any matchup, switch between four prediction models, run what-if scenarios, see a 100,000-run tournament outlook, and read an AI analyst brief written in a football-pundit voice.

The frontend runs entirely on **pre-computed static JSON** exported from an offline Python ML pipeline — no training, no Python, no `.joblib` files are loaded at request time. OpenAI is used **only** for short cached explanations and never overrides model probabilities.

## Overview

Pick two teams, pick a match context (Group Stage → Final), pick a model lens, and the app returns a calibrated win/draw/win probability split, an expected scoreline, the key tactical factors, and a football-language translation of those factors. A separate panel below shows the projected tournament-wide outlook from 100,000 offline Monte Carlo simulations of the 2026 bracket.

Built as a portfolio piece to demonstrate end-to-end ML productisation: offline training in Python → static artifact export → typed TypeScript inference layer in the browser → LLM-powered explanation overlay → real-world deployment on Vercel.

## Key features

- **Four prediction models**, selectable from the sidebar:
  - **Calibrated ML** *(default)* — CatBoost classifier trained on 2018–2026 senior international results, with isotonic probability calibration.
  - **Legacy Historical** — the original transparent heuristic baseline built on World Cup history.
  - **ML Benchmark** — a Logistic Regression comparison model.
  - **Elo + Score** — Elo-style ratings combined with a Poisson expected-goals scoreline model.
- **Modern Squad Strength toggle** *(off by default)* — applies a small curated squad-strength adjustment after the selected base model.
- **Monte Carlo tournament outlook** — 100,000 offline simulations of the full 2026 bracket. Champion, finalist, semifinal, quarterfinal, and round-of-32 probabilities per team.
- **AI Analyst Brief** — five-card brief (Key Takeaway, Why the Favorite is Favored, Risk Factor, Upset Path, Model Limitation) generated by OpenAI in a premium football-pundit tone. Cached server-side.
- **Football Translation panel** — translates raw factors (Elo gap, expected goals, recent form, venue, model lens) into plain football language alongside the Key Factors list.
- **What-if lab** — sliders for recent form, attack, defensive stability, and a knockout-pressure toggle.
- **Host City Energy** — 16 venues with poster art, fixtures, and one-click "load this match" buttons.
- **2026 Scenario view** — stage-by-stage bracket simulator with custom matchups.
- **Validated metrics panel** — shows accuracy, macro F1, and a historical-baseline comparison for the active model.

## Model architecture

```
                  OFFLINE                                    ONLINE
┌─────────────────────────────────────┐         ┌──────────────────────────┐
│  Python ML pipeline (ml/)           │         │  Next.js frontend (src/) │
│  ┌───────────────────────────────┐  │  JSON   │                          │
│  │ CatBoost (calibrated)         │──┼─────────┤  Calibrated ML lens      │
│  │ Logistic Regression           │──┼─────────┤  ML Benchmark lens       │
│  │ Elo + Poisson scoreline       │──┼─────────┤  Elo + Score lens        │
│  │ Historical heuristic baseline │──┼─────────┤  Legacy Historical lens  │
│  │ Monte Carlo (100k runs)       │──┼─────────┤  Tournament outlook      │
│  └───────────────────────────────┘  │         │                          │
│                                     │         │  /api/explain  ──► OpenAI│
└─────────────────────────────────────┘         │  (analyst brief only)    │
                                                └──────────────────────────┘
```

Single-match predictions read from `ml/model_outputs/matchup_predictions.json`. The Monte Carlo outlook reads from `ml/model_outputs/worldcup_simulation.json`. The frontend never touches a Python process, a `.joblib` artifact, or raw training data at runtime. Monte Carlo outputs are used **only** for the tournament view — they never override single-match probabilities.

### Files in `ml/`

- `simulate_tournament.py` — self-contained, stdlib-only Monte Carlo simulator (`python3 ml/simulate_tournament.py --runs 100000 --seed 42`).
- `data/worldcup_2026/worldcup.json` — 2026 fixture used by the simulator.
- `model_outputs/` — static JSON consumed by the Next.js app:
  - `matchup_predictions.json` — pre-computed matchup probabilities across all models.
  - `worldcup_simulation.json` — Monte Carlo outlook (champion / finalist / SF / QF / R32 probabilities per team).
  - `team_strengths.json`, `elo_history.json`, `model_metrics.json`, `feature_importance.json`, `probability_calibration.json`, `prediction_examples.json`, `shap_explanations.json`, `team_features.json`.
- `train_model.py` — earlier-generation in-repo trainer (Logistic Regression / Random Forest / Calibrated Gradient Boosting / Elo + Poisson).
- `squad_strength/` — curated CSV + script that produces the optional Modern Squad Strength adjustment layer.

The **CatBoost training pipeline** lives in this repo at `ml/ml-pipeline/`. It is a self-contained Python project that ingests historical international match results, builds Elo + rolling-form features, trains and evaluates CatBoost / LightGBM / XGBoost / Logistic Regression / Random Forest / Elo+Poisson, calibrates the winning model, and exports the JSON artifacts that ship in `ml/model_outputs/`. See `ml/ml-pipeline/README.md` and `ml/ml-pipeline/requirements.txt` for setup. Trained model `.joblib` artifacts, the `.venv`, charts, and raw data are intentionally gitignored — re-running the pipeline regenerates them locally.

## Tech stack

- **Frontend:** Next.js 16 (Turbopack), React 19, TypeScript 5
- **Styling:** Hand-rolled CSS in `src/app/globals.css` (custom pastel theme, no Tailwind)
- **Linting:** ESLint 9 with `eslint-config-next`, `--max-warnings=0`
- **AI explanations:** OpenAI Node SDK, server-side route at `src/app/api/explain/route.ts`, cached in memory by matchup + model + version
- **Offline ML pipeline:** Python 3, pandas, NumPy, scikit-learn, CatBoost (in the sibling `ml-pipeline/` project)
- **Monte Carlo simulator:** Python 3 stdlib only (`ml/simulate_tournament.py`)
- **Hosting:** Vercel (recommended) — the frontend is fully static-importable; only `/api/explain` runs server-side.

## Local setup

Prerequisites: Node 20+, npm, optionally Python 3.9+ if you want to regenerate Monte Carlo data.

```bash
git clone https://github.com/<your-username>/world-cup-predictor-lab.git
cd world-cup-predictor-lab
npm install
npm run dev
```

Open http://localhost:3000.

Useful scripts:

```bash
npm run lint      # ESLint
npm run build     # Production build (Turbopack)
npm run start     # Run the production build
npm run generate:data  # Regenerate src/data/world-cup-data.json from public/data
```

Regenerate the Monte Carlo outlook (100k runs, ~40 s):

```bash
python3 ml/simulate_tournament.py --runs 100000 --seed 42
```

Regenerate the Modern Squad Strength export:

```bash
python -m pip install -r requirements.txt
python ml/squad_strength/build_squad_strength.py
```

The frontend builds and runs **without any API key** — the AI Analyst Brief simply falls back to short football-toned default copy.

## Environment variables

To enable the OpenAI-powered analyst brief, create a `.env.local` file at the project root (this file is already gitignored — never commit it):

```
OPENAI_API_KEY=sk-...your-key-here...
OPENAI_MODEL=gpt-5-nano
```

`OPENAI_MODEL` is optional and defaults to `gpt-5-nano` when omitted. The key is read server-side only inside `src/app/api/explain/route.ts` and never exposed to the browser bundle.

## Deployment (Vercel)

1. Push the repo to GitHub.
2. Go to https://vercel.com/new and import the GitHub repo.
3. Framework preset: Next.js (auto-detected).
4. Under **Environment Variables**, add both:
   - `OPENAI_API_KEY` → your key
   - `OPENAI_MODEL` → `gpt-5-nano`
5. Click **Deploy**.

Vercel automatically picks up the `next.config.ts`, `package.json`, and TypeScript settings. The pre-computed JSON artifacts in `ml/model_outputs/` are bundled at build time — no runtime data fetch from any external source.

## Disclaimer

This project is built for **portfolio, educational, and analytical purposes**. All probabilities shown in the app are model-estimated probabilities — not betting odds, not predictions of certain outcomes, and not investment or wagering advice. Football is decided on the pitch by single moments the model cannot see. Do not use this app to make betting decisions.

The Modern Squad Strength layer uses manually curated proxy values (FIFA points, market value, caps, club strength) for a limited sample of teams; it is a directional indicator, not a player-level production model. AI analyst briefs are generated by an LLM and may occasionally state things inaccurately — they are explanatory commentary on the model's read, not factual reporting.
